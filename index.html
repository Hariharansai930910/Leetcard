704. Binary Search

Given an array of integers nums which is sorted in ascending order, and an integer target, write a function to search target in nums. If target exists, then return its index. Otherwise, return -1.

You must write an algorithm with O(log n) runtime complexity.


Example 1:

Input: nums = [-1,0,3,5,9,12], target = 9
Output: 4
Explanation: 9 exists in nums and its index is 4

Example 2:

Input: nums = [-1,0,3,5,9,12], target = 2
Output: -1
Explanation: 2 does not exist in nums so return -1
Hints for Binary Search
  • The key idea is to divide the search space in half each time
  • Use two pointers: left and right to track the search space
  • Calculate the middle index and compare with target value
  • If target is less than the middle value, search in the left half
  • If target is greater than the middle value, search in the right half
  • If target is equal to the middle value, return the middle index
  • Be careful about integer overflow when calculating the middle index
  • Continue until either the target is found or the search space is empty
Solution Blueprint
1. Initialize left pointer to 0 and right pointer to length of array - 1
2. While left <= right:
   a. Calculate middle index: m = left + (right - left) // 2
   b. If nums[m] > target:
      i. Search in left half: right = m - 1
   c. Else if nums[m] < target:
      i. Search in right half: left = m + 1
   d. Else:
      i. Target found: return m
3. If loop exits without finding target, return -1
Time: O(log n) Space: O(1)
Optimized Implementation
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        l, r = 0, len(nums) - 1
        while l <= r:
            # (l + r) // 2 can lead to overflow
            m = l + ((r - l) // 2)  
            if nums[m] > target:
                r = m - 1
            elif nums[m] < target:
                l = m + 1
            else:
                return m
        return -1
Time: O(log n) Space: O(1)
BINARY SEARCH
Memory Aids
  • L.M.R Pattern: Left, Middle, Right pointers to divide and conquer.
  • Half & Half: Always eliminate half the search space in each iteration.
  • Safety First: Use m = l + (r - l) // 2 to avoid integer overflow.
  • Equal Sign Matters: Use l <= r to ensure single element arrays are processed.
  • Comparison Triangle:
    • nums[m] > target: move right pointer left
    • nums[m] < target: move left pointer right
    • nums[m] == target: found it!
  • Think Logarithm: If the input size halves each time, it's likely O(log n).
